元素类型（块级元素，内联元素，内联块级元素）

块级元素

  • 块级元素在网页中就是以块（元素显示为矩形区域）的形式显示
    • 常见的块级元素： div, dl, dt, dd, ol, ul, li, h1~h6, p, form, hr, colgroup, fieldset, col, table, tr, td
  • 默认情况，块级元素都会 占一行 ，通俗地说，两个相邻块级元素不会出现 并列显示 的现象，默认情况下，块级元素会按照顺序自上而下排列。
  • 块状元素都可以定义自己的高度和宽度，不设置宽度的时候，默认和 父级元素 一样宽

比如说只有一个 <p> 标签在 <body> 里面的话它的父级元素就是 <body> ，和body一样宽

  • 块状元素一般都作为其他元素的 容器 ，它可以 容纳其他内联元素和其他块状元素 。我们可以把这种元素比喻为一个盒子

内联元素

  • 常见的内联元素如： a, span, i, em, strong, b 等
  • 内联元素的表现形式是始终以 行内逐个进行显示
  • 内联元素没有自己的形状， 不能 定义它的 宽和高 ，它显示的宽度、高度只能根据所包含内容的宽度和高度来确定，它的最小内容单元也会呈现矩形形状
  • 内联元素也会遵循 盒模型 基本规则，如可以定义 padding, border, margin, background 等属性，但个别属性不能正确显示，对于盒子模型属性 部分支持

内联块级元素（inline-block）

  • 内联块（行内块）元素是块级元素和内联元素的混合体，inline-block元素设置宽度和高度
  • 内联块级元素属于 内联元素 ， 不会独占一行 ，而是会和其他inline元素和inline-block元素在同一行按 从左至右的顺序显示
  • 常见的内联块级元素： img, textarea, input, select, iframe 等

三种元素之间的转换

  • 可以将元素的 display 属性更改，将其转换类型为 block, inline, inline-block 中的一种

注意：如果想要让一个块级元素在一行内显示，又想能够设置宽和高，要将其设置为 内联块级元素

选择器的运用

选择器的类型

元素选择器/标签选择器（element选择器）

  • 语法：
1

 元素名称{属性：属性值；}
  • 用法：
    • 如果想改变某个元素的默认样式时，可以使用 元素选择器
    • 统一文档某个元素显示效果时，可以使用 类型选择器

id选择器

  • 每一个元素都可以有一个自己的 id 属性，但是 一个页面 中每个元素的 id 值全都 不能相同
  • id选择器的语法格式： "#"加上自定义的id名称

如 #top{width:300px;height:300px;}

  • id名命名取英文名，不能使用关键字（所有的标记和属性都是关键字），如 head标记
  • 一个id名称只能对应文档中的一个具体元素对象，因为id只能定义页面中某一个唯一的元素对象
  • id选择器最大的好处：创建网页的外围结构

class选择器

  • 多个标签可以同属于一个 class ，在设置了这个class的属性之后，所有属于这个class的标签都会被设置成同一个属性
  • 用法：
1
2

.top{width:200px;height:500px;}  /*设置top这个class的属性*/
<div class="top"></div>

注意： 类名的第一个字符 不能使用数字 ，它无法在Mozilla或Firefox中起作用

*通配符（全局选择器）

  • *{} 这个选择器表示所有的元素都会遵循这个规则，一般写在最前面
  • 一般写css的时候会在开头写上
1
2
3
4
5

* {
    padding: 0; 
    margin: 0;  
   /*其实都是(0px),不过在写0的时候可以不用写px*/
}

交集选择器

  • 语法：
1

选择器1选择器2{属性：属性值；}
  • 用于选择同时有多个选择器匹配的元素
  • 多个选择器中间不能写空格

并集选择器（群组选择器）

  • 语法：
1

选择器1，选择器2，选择器3{属性：属性值；}
  • 当有多个选择器 应用相同的样式 时，可以将选择器用 , 分隔，合并为一组。

  • 语法：

  • 说明：选择器1和选择器2用空格隔开，含义就是选择器1中 包含的所有选择器2 （只要是它的后代都会变颜色，不管是子几代）

选择器的权重

  • 在css中，会根据选择器的特殊性来决定所定义的样式规则的次序，具有更特殊选择器的规则 优先于 具有一般选择器的规则，如果两个规则的特性相同，那么后定义的规则优先。
  • css中用四位数字表示权重，权重的表达方式如: 0, 0, 0, 0
    • 标签选择器 的权重为 0001
    • class选择器 的权重为 0010
    • id选择器 的权重为 0100
    • 属性选择器 的权重为 0010
    • 伪类选择器 的权重为 0010
    • 伪元素选择器 的权重为 0010
    • 包含选择器 的权重为： 包含选择器的 权重之和
    • 子选择器 的权重为 0000
    • 群组选择器 的权重为 0000
    • 继承样式 的权重为 0000
    • 行内样式 的权重为 1000

盒模型

盒的内容

  • overflow : overflow属性指定如果内容 溢出 一个元素的框，会发生什么
    • overflow-x 规定如果x轴溢出了会怎么样， overflow-y 规定如果y轴溢出了会怎么样
  • text-overflow 属性和 overflow 属性类似，但是只规定文本的属性，overflow属性则规定整个元素(整个div)
    • 一定要注意需要使用 text-overflow 属性的时候其必须和 loverflow:hidden;white-space:nowrap 配合使用

盒的边框

  • 元素的边框（border）是围绕元素内容和边框的一条或多条线，css border属性允许规定边框的样式，宽度和颜色

如果不设置 *{margin:0;padding:0;} 的话， margin 是默认有值的，默认上下左右都是 8

padding

  • 是内容和边框之间的填充，用来调整内容在盒子中的位置
  • 盒子和盒子之间的距离不是累加的和，而是最大值，如果上面一个div的 margin-bottom 是 70px ，下面一个div的 margin-top 是 50px ，那么上下两个div之间的间距不是120px而是 70px

margin

  • margin 的属性和 padding 的属性大同小异， bottom, top, left, right

  • <fieldset> 标签不常用，放在里面的东西会被框在一起，里面的 <legend> 标签可以设定这个框框的标题
  • <input type="file" /> 用来上传文件，其 multiple 属性可以用来设置是否可以上传多个文件

浮动

  • 文档流基本上可以算作是 固定在页面上面的 ，而浮动流则是可以在文档流上面 飘来飘去 的，其实在看浮动流的时候可以当作文档流不存在
  • 如果浮动 非替换 元素，则要指定一个明确的宽度，否则，它们会尽可能地窄 替换元素和非替换元素
  • 假设某个div的元素 A 是浮动的，如果A元素的上一个元素也是浮动的，那么A元素会 跟随在上一个元素 的 后面 ，但是如果一行放不下这两个元素，那么A元素会被 挤到下一行
  • 浮动元素不在标准文档流中，所以浮动后面紧跟的元素占据了浮动元素原先的位置
  • 要想避免浮动出现的一些问题，可以清除浮动，即不允许左边或右边出现浮动元素

css定位

  • css定位（position）属性允许你定义元素 相对于其正常位置 应出现的位置，或者相对于 父元素 ， 另一个元素 甚至 浏览器窗口本身的位置
  • css定位机制：普通流，浮动流，绝对定位
  • 绝对定位(position:absolute)
  • 固定定位

z-index 属性

  • z-index 属性：设置元素的 层叠顺序 ，属性值为无单位的整数值，值较大的元素会叠加在值较小的元素的上面
